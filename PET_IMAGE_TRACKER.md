## Naming Model

State images use:

`<stage>_<state>.png`

Stages:
- `eggling`
- `baby`
- `child`
- `teen`
- `adult`
- `elder`

State variants:
- `healthy`
- `hungry`
- `poopy`
- `dirty`
- `restless`
- `hungry_poopy`
- `hungry_dirty`
- `hungry_restless`
- `poopy_dirty`
- `poopy_restless`
- `dirty_restless`
- `hungry_poopy_dirty`
- `hungry_poopy_restless`
- `hungry_dirty_restless`
- `poopy_dirty_restless`
- `hungry_poopy_dirty_restless`
- `critical`
- `dead`

Sticker derivatives use:

`<stage>_<state>.webp`

and live in [pet_stickers](/home/phill/projects/telegram-archive-bot/pet_stickers).

## Action Image Model

Important correction:

The pet action images are not mostly one-off generic event assets.

For pet-facing actions, the image itself should include the pet, which means those actions need a separate image for each life stage.

So the action images should follow:

`action_<action>_<stage>.png`

Examples:
- `action_feed_eggling.png`
- `action_feed_baby.png`
- `action_feed_child.png`
- `action_feed_teen.png`
- `action_feed_adult.png`
- `action_feed_elder.png`
